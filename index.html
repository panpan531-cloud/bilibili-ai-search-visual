01 / USER SIGNAL

半年反馈不是单点故障，而是搜索信任的连续磨损

小红书半年样本中，抱怨集中在“搜词与结果不匹配”“搜索页像推荐页”“已看/不喜欢/黑名单仍被推送”“老视频和官方内容搜不到”。正向信号也存在：用户仍信任 B站的内容资产、弹幕和创作者，只是入口体验让人怀疑。

1月

“所有人的 B站都坏掉了”

高互动吐槽已经出现：搜索结果一半以上无关，相关视频、关注内容、黑粉视频被混入。

3-4月

搜索页被感知为推荐页

“猜你喜欢”进入搜索结果，用户怀疑搜索推荐机制、投流或商业化影响排序。

5月

精确搜索失败被放大

用户明确描述“只有一半和搜索词有关”，甚至站内搜不到、外部搜索能找到站内视频。

6月

问题成为公共吐槽话题

“B站搜索没救了”“完全搜不到”这类笔记持续出现，评论区补充大量场景案例。

负向 1

相关性被推荐污染

用户打开搜索是带着任务来的，结果看到近期热点、平时观看、擦边/黑粉内容，会直接破坏“搜索中立性”。

负向 2

精确找视频失败

已知标题、官方资源、老视频、片段记忆都可能找不到，用户转向百度、夸克、微博或收藏夹。

负向 3

控制感失效

“不感兴趣”、黑名单、排序筛选、退出登录等绕路办法被反复提及，说明用户想夺回结果控制权。

正向

内容资产仍被认可

用户不是不想用 B站，而是不相信入口。AI 搜的机会是让优质 PUGV、弹幕、评论和合集重新可发现。

02 / QUERY TAXONOMY

先把 query 看成“用户任务”，再谈 AI 怎么接住

下面是结合通用搜索意图研究、视频搜索特征与 B站内容生态推导的工作假设。它不是声称掌握 B站真实日志，而是一个可用于产品决策讨论、再用 query log 校准的分布模型。

100% Search Tasks

分布基于 Broder/Jansen 的搜索意图框架、视频搜索 intent 分类论文，以及 B站 PUGV/社区消费场景的产品推导。

Selected intent

精确找视频 / 已知目标

用户知道自己要找什么：标题、UP主、番剧/影视资源、某个教程合集。AI 不应替用户“聊天”，而应更强地做纠错、别名识别、站内实体归并和精准排序。

标题纠错官方资源优先老视频召回
03 / AI SEARCH SCENARIOS

AI 搜不等于“搜索框里放大模型”，而是按意图重构结果形态

最适合 B站的 AI 搜，是把视频、字幕、弹幕、评论、合集、UP主关系和用户控制信号一起接入，让不同 query 得到不同结果页。

AI Search Result Page
用户 query 那个讲“工业糖精”的影视吐槽片段
AI 判断：模糊记忆找片段

AI 精准找视频 / 片段定位

从标题、ASR字幕、OCR画面字、弹幕热词和评论中召回候选，给出可跳转时间点，而不是只返回整条视频列表。

08:42 UP主A：影视剧里的工业糖精套路

字幕命中“工业糖精”，弹幕在 08:42-09:13 密集讨论。

13:05 UP主B：甜宠剧为什么让人出戏

评论区高频提到“工业糖精”，相似语义命中。

用户价值

减少“明明看过但搜不到”的挫败感，把搜索从列表翻页变成定位。

平台价值

提升老视频、长视频和优质 PUGV 的可发现性，增加有效播放与收藏。

关键能力

ASR/OCR/弹幕评论索引、语义召回、时间点证据、结果解释。

04 / PRODUCT BET

核心主张：先做“AI 精准找视频/片段 + AI 教程攻略”

这两个场景兼顾用户痛点强度、B站独特数据资产和可衡量闭环。它们不会把 B站变成通用问答工具，而是让 B站已有内容更容易被找到、被理解、被消费。

05 / MVP ROADMAP

落地不是“大模型炫技”，而是搜索质量系统升级

从低风险入口开始，让用户看见 AI 如何参与召回、排序和总结，并保留足够控制权。

0-1 月

意图分类与结果页分流

识别已知目标、模糊找片段、教程攻略、测评对比等 query，决定是否触发 AI 卡。

  • 搜索日志标注抽样
  • query rewrite / 纠错
  • 推荐混入强约束
1-3 月

AI 片段定位 Beta

接入字幕、OCR、弹幕、评论语义索引，对高置信结果给时间点和证据。

  • 时间点跳转
  • 为什么命中解释
  • 低置信不强行回答
3-6 月

教程攻略聚合

把多条视频总结为步骤、坑点、先看后看路径，并把播放导回原视频。

  • 引用来源卡
  • 系列/合集整合
  • 学习路径收藏
06 / MEASUREMENT

成功指标要证明“搜得准、看得下、信得过”

Search Success Rate ↑

搜索后有效点击、播放超过阈值、收藏/投币/关注等组合成功信号。

Reformulation Rate ↓

同一会话内反复改 query 的比例下降，尤其是精确找视频场景。

Timestamp CTR ↑

AI 时间点卡是否真的帮助用户进入内容内部，而不是停在摘要。

Trust Feedback ↑

“结果无关”“不想看还推”“搜不到”类负反馈下降。

07 / GUARDRAILS

AI 搜的底线：不要制造新的搜索不信任

幻觉风险

AI 卡必须引用视频、时间点、字幕/弹幕/评论证据；低置信场景只做召回增强，不生成肯定结论。

创作者流量风险

摘要不能替代观看，关键交互要导向原视频、合集和UP主主页。

推荐污染风险

搜索页要明确区分“搜索相关结果”和“探索推荐”，给用户开关与解释。
